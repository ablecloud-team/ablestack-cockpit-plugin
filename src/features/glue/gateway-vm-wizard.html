ABLESTACK 게이트웨이 VM 배포 마법사

ABLESTACK 클러스터에 Glue 공유스토리지 게이트웨이를 실행하는 가상머신 어플라이언스를 배포합니다.

ABLESTACK 게이트웨이 VM을 배포하기 위해 필요한 정보를 다음과 같이 마법사를 통해 입력받습니다.

  • 게이트웨이 VM의 HA구성을 위한 클러스터 설정 정보
  • 게이트웨이 VM의 CPU, Memory, Disk, Network에 대한 정보
  • 가상머신의 호스트명 등의 네트워크 정보
  • 호스트 및 가상머신 간의 상호 SSH 연결을 위한 SSH Key 정보

필요한 정보를 먼저 준비하십시오. 정보가 준비되었다면 "다음" 버튼을 눌러 게이트웨이 VM 배포를 시작합니다.

Info alert:컴퓨트 자원 구성 시 참고사항

게이트웨이 VM의 Compute 자원은 Glue 공유스토리지 게이트웨이 가상머신이 관리해야 할 서비스 수에 따라 탄력적으로 선택합니다.

게이트웨이 가상머신이 컨트롤 할 스토리지 서비스 수가 2개 미만이면 4 vCore를, 그 이상이면 8 vCore를 선택하십시오. 가상머신이 컨트롤 할 스토리지 서비스 수가 2개 미만이면 8GiB를, 그 이상이면 16GiB를 선택하십시오.

ROOT Disk의 크기는 70GiB를 디스크가 Thin Provisioning 방식으로 제공됩니다.

Info alert:네트워크 구성 시 참고사항

게이트웨이 기싱머신 스토리지 서비스에 접근하고자 하는 네트워크 위치에 따라 가상머신에 할당할 네트워크와 네트워크의 상위 브릿지를 선택합니다.

관리네트워크는 호스트를 관리하기 위해 필수적인 네트워크이며, 기본 선택되어 있고, 선택 여부를 변경할 수 없습니다. 서비스네트워크는 사용자가 접근하고자 하는 네트워크 위치이며, 관리네트워크와 서비스네트워크가 다른 경우 선택합니다.

가상머신에 네트워크를 할당하기 전에 반드시 상위 브릿지를 먼저 생성해야 합니다.

게이트웨이 VM에 호스트명 등의 부가적인 네트워크 정보를 설정합니다. 아래의 항목에 값을 입력하십시오.

게이트웨이 VM과 호스트, 그리고 ABLESTACK을 구성하고 있는 가상머신들과의 SSH 연결을 위해 SSH Key를 설정합니다. 기본적으로 현재 호스트의 SSH Key 파일을 자동으로 등록하며, 필요시 다운로드 한 SSH Key 파일로 등록 가능합니다.


Info alert:SSH Key 등록 참고사항

SSH Key는 호스트 및 게이트웨이, 클라우드센터, 스토리지센터 가상머신 등의 ABLESTACK 구성요소 간의 암호화된 인증을 위해 사용됩니다.

호스트 간, 가상머신 간의 모든 명령은 SSH를 이용해 전달되며 이 때 SSH Key를 이용해 인증을 처리합니다. 따라서 모든 호스트, 가상머신은 동일한 SSH Key를 사용해야 합니다.
